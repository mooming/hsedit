# PLAND.md — MakeBuilder Integration

## Goal
Configure hsedit to use makebuilder for automatic CMake generation. makebuilder scans directory structure + `.module.config` files and generates `CMakeLists.txt` automatically.

## Steps

### 0. makebuilder as submodule ✅
- `external/makebuilder` is now a git submodule

### 1. Build makebuilder ✅
- `./external/makebuilder/build.sh -release`
- Produces binary at `build/Application/MakeBuild/Release/makebuild`

### 2. Restructure project layout ✅
- `src/main.cpp` → `app/main.cpp`
- `src/` → StaticLibrary (module_registry, event_bus)
- `app/` → Executable (main entry point)
- Added `modules/.module.config` (None) to enable traversal into `modules/terminal/`
- Added `src/tutorial/.module.config` (Ignored) to skip stale tutorial

### 3. Create configuration files ✅
- `.project.config` — C++17, compile flags
- `.module.config` (root) — buildType = None
- `customCMake.txt` — global ncurses find_library + imported target
- `src/.module.config` — StaticLibrary, name = libhsedit
- `app/.module.config` — Executable, name = hsedit, dependency = libhsedit
- `modules/terminal/.module.config` — Executable, library = ncurses
- `tests/.module.config` — Executable, name = hsedit_tests, dependency = libhsedit, library = ncurses

### 4. Run makebuilder ✅
- `./external/makebuilder/run.sh /Users/anav/atelier/hsedit`
- Generated CMakeLists.txt for all 6 modules

### 5. Build and verify ✅
- All three targets build in Release, Dev, Debug configs
- `hsedit` runs correctly, `terminal_module` runs correctly, `hsedit_tests` passes all tests
- ncurses linked via imported target from root customCMake.txt
- `lib/` added to .gitignore
- README.md updated with full build workflow
- Committed as `6f004ba`
