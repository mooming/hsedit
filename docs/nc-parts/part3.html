7. Colors

Colors in ncurses require initialization. Here's the pattern:

// 1. Start color mode
start_color();

// 2. Define color pairs (foreground, background)
// Pair 1: white text on blue background
init_pair(1, COLOR_WHITE, COLOR_BLUE);
// Pair 2: yellow on black (status bar)
init_pair(2, COLOR_YELLOW, COLOR_BLACK);
// Pair 3: green on black (success)
init_pair(3, COLOR_GREEN, COLOR_BLACK);

// 3. Use a color pair
attron(COLOR_PAIR(1));
printw("White on blue");
attroff(COLOR_PAIR(1));

// Or combine with attributes
attron(A_BOLD | COLOR_PAIR(2));
printw("Bold yellow on black");
attroff(A_BOLD | COLOR_PAIR(2));

Limitation
Standard ncurses supports only 256 color pairs. Each pair is a foreground/background combination. For more colors, you need the ncursesw variant with use_default_colors().

8. Keypad & Special Keys

The keypad() function you call during initialization tells ncurses to translate escape sequences from special keys (arrows, F-keys) into symbolic constants.

// MUST be called on stdscr during initialization
keypad(stdscr, TRUE);

Without keypad(stdscr, TRUE), pressing the Up arrow gives you raw escape sequences (ESC [ A) instead of KEY_UP.

Critical
Always call keypad(stdscr, TRUE) or your editor won't respond to arrow keys, function keys, or any special key. This is the most common beginner error.

9. Echo, Cursor & Visibility

Echo Control

noecho();     // Don't show typed characters (default after noecho)
echo();       // Show typed characters as you type

For an editor, you almost always want noecho(). For a command-line input (like a search bar), you temporarily enable echo.

Cursor Control

 curs_set(1);     // Visible cursor (0=hidden, 1=block, 2=underline)
move(y, x);       // Move cursor to (y, x)

Nodelay & Timeout

// nodelay: getch returns ERR immediately if no key
nodelay(stdscr, TRUE);

// timeout: getch blocks for ms milliseconds, then returns ERR
timeout(50);      // 50ms timeout

// Blocking (default)
nodelay(stdscr, FALSE);
timeout(-1);       // Negative = infinite wait

10. Event Loop Pattern

Here's the standard pattern for a TUI editor's main loop:

// After initscr() and configuration...

bool running = true;
while (running) {
    // 1. Render the current state
    draw_screen();

    // 2. Get input (non-blocking for polling, or blocking for waiting)
    timeout(-1);     // Block until key press
    int key = getch();

    // 3. Handle the key
    switch (key) {
        case KEY_UP:
            move_cursor_up();
            break;
        case KEY_DOWN:
            move_cursor_down();
            break;
        case 'q':
            running = false;
            break;
        case KEY_RESIZE:
            handle_resize();
            break;
        // ... more key handlers ...
    }
}

endwin();
