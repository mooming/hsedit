11. Terminal Resize

ncurses sends KEY_RESIZE when the user resizes the terminal. You should handle this:

// In your event loop:
case KEY_RESIZE:
    handle_resize();
    break;

// Resize handler:
void handle_resize() {
    // Wipe and recreate all windows to new size
    delwin(status_bar);    // Delete old windows
    delwin(editor_win);
    erase();

    // Recreate with new dimensions
    int rows = LINES;
    int cols = COLS;
    status_bar = newwin(1, cols, rows - 1, 0);
    editor_win = newwin(rows - 2, cols, 0, 0);
}
Tip
On macOS, the terminal may also send a SIGWINCH signal. You can install a signal handler to set a flag, which your event loop checks alongside KEY_RESIZE.

12. Cross-Platform Notes

Windows Considerations

  • MSYS2/MinGW provides ncurses. Compile with the MinGW g++ from MSYS2.
  • Windows terminals have different defaults (no mouse support in older terminals).
  • wgetch() and waddwstr() are the wide-character equivalents for Unicode.
  • Line endings: Windows uses \r\n. ncurses handles this, but be aware when reading/writing files.

macOS Considerations

  • ncurses is included with Xcode command line tools. No extra install needed.
  • Use brew install ncurses only if you need features not in the system version.

Linux Considerations

  • Package names vary: libncurses-dev (Debian/Ubuntu), ncurses-devel (Fedora/RHEL).
  • Some systems have both libncurses and libncursesw. Prefer -lncursesw for Unicode.
Tip
Use a preprocessor guard to detect the platform if you need platform-specific behavior: 
#ifdef _WIN32
    // Windows-specific code
#elif __APPLE__
    // macOS-specific code
#elif __linux__
    // Linux-specific code
#endif

Key Design Decisions for Your Editor

  1. Use wide-character functions: wgetch(), waddwstr(), wprintw() for Unicode support.
  2. One window for stdscr: Don't create too many sub-windows. Use mvprintw on stdscr directly for simplicity.
  3. Separate render from logic: Your command logic shouldn't know about ncurses. Use an abstraction layer.
  4. Handle KEY_RESIZE: Always handle terminal resize to avoid display corruption.
  5. Use timeout(-1) for blocking input in your main loop, or timeout(50) for polling if you need periodic updates.

Minimal Editor Skeleton

#include <ncurses.h>
#include <vector>
#include <string>

// Text buffer (simple line-based for now)
struct Buffer {
    std::vector<std::string> lines;
    int cursor_y = 0;
    int cursor_x = 0;
};

// Render the buffer to the terminal
void render(const Buffer& buf) {
    erase();
    int rows = LINES - 1;  // Reserve last line for status
    for (int i = 0; i < rows; ++i) {
        if (i < buf.lines.size()) {
            mvprintw(i, 0, "%s", buf.lines[i].c_str());
        }
    }
    // Status bar
    mvprintw(rows, 0, "Lines: %d | Cursor: %d,%d",
        (int)buf.lines.size(), buf.cursor_y, buf.cursor_x);
    refresh();
}
