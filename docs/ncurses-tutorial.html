ncurses Tutorial

A practical guide for building a TUI text editor in C++

1. Overview & Philosophy

ncurses (new curses) provides a terminal-independent API for creating text-based user interfaces. It abstracts away differences between Linux, macOS, and Windows terminals.

Key Concepts

  • Raw mode: Characters you type aren't echoed — your program decides what to display.
  • Windows: Rectangular terminal regions. stdscr is the full screen.
  • Coordinates: (y, x) — row first, then column. This is the #1 beginner mistake.
  • Buffered output: ncurses draws to an internal buffer. Call refresh() to display at once (prevents flicker).
Why ncurses?
Available everywhere. Lightweight. Mature (30+ years). Small enough API to learn in a day.

Use -lncursesw for Unicode. The w means "wide character."

3. Your First Program

Every ncurses program follows the same skeleton: initscr() → configure → loop → endwin().

#include <ncurses.h>

int main() {
    // 1. Initialize ncurses
    initscr();

    // 2. Configure terminal
    cbreak();          // Each key press arrives immediately (no Enter needed)
    noecho();          // Don't echo typed characters automatically
    keypad(stdscr, TRUE);  // Enable special key codes (arrows, F1...)

    // 3. Write text
    mvprintw(0, 0, "Hello, ncurses!");

    // 4. Force display
    refresh();

    // 5. Wait for any key
    getch();

    // 6. Restore terminal state
    endwin();
    return 0;
}
Always call endwin()
If your program crashes without endwin(), your terminal is left broken (no echo, no line editing). Always call it.

4. Windows & Coordinates

A window in ncurses is a rectangular drawing area. Think of it as a canvas within the terminal.

Creating Windows

// stdscr — full terminal screen (created automatically)
// newwin(rows, cols, start_y, start_x)
WINDOW* win = newwin(10, 40, 2, 5);

// Write to the sub-window
wprintw(win, "Inside sub-window");
wrefresh(win);

Coordinate System

mvprintw(y, x, "text") — y first, then x.

Querying Dimensions

int rows   = LINES;
int cols   = COLS;
int cur_y  = getcury(stdscr);
int cur_x  = getcurx(stdscr);

5. Input Handling

Basic getch()

// Blocking: waits for a key press
int key = getch();

// Non-blocking: returns ERR (-1) if no key pressed
nodelay(stdscr, TRUE);
int key = getch();   // Returns ERR if nothing pressed

Common Key Codes

ConstantMeaning
KEY_UPUp arrow
KEY_DOWNDown arrow
KEY_LEFTLeft arrow
KEY_RIGHTRight arrow
KEY_HOMEHome key
KEY_ENDEnd key
KEY_BACKSPACEBackspace
KEY_DCDelete key
KEY_ICInsert key
KEY_ENTEREnter/Return
KEY_RESIZETerminal resized
KEY_F(n)F1..F12 (e.g. KEY_F(1))
KEY_MOUSEMouse event

Modifiers (Ctrl, Shift, Alt)

// Ctrl combinations are single bytes:
int key = getch();
if (key == 3)  // Ctrl+C (ASCII 3)
if (key == 21) // Ctrl+U
if (key == 19) // Ctrl+S  (common: save)
if (key == 1)  // Ctrl+A  (common: command mode)

// Alt + character: ncurses sends ESC first, then the character
if (key == 27) {  // ESC pressed — check if followed by a char
    nodelay(stdscr, TRUE);
    int next = getch();
    if (next != ERR) {
        // Alt+key: ESC + character = 27 + next
        handle_alt_key(next);
    }
}

Input Buffers

// Clear the input buffer (discard pending keystrokes)
flushinp();

// Set a timeout — getch returns ERR after ms milliseconds
timeout(100);    // 100ms timeout
halfdelay(5);   // 0.5 second timeout (halfdelay mode)

7. Colors


Colors in ncurses require initialization. Here's the pattern:


// 1. Start color mode
start_color();

// 2. Define color pairs (foreground, background)
// Pair 1: white text on blue background
init_pair(1, COLOR_WHITE, COLOR_BLUE);
// Pair 2: yellow on black (status bar)
init_pair(2, COLOR_YELLOW, COLOR_BLACK);
// Pair 3: green on black (success)
init_pair(3, COLOR_GREEN, COLOR_BLACK);

// 3. Use a color pair
attron(COLOR_PAIR(1));
printw("White on blue");
attroff(COLOR_PAIR(1));

// Or combine with attributes
attron(A_BOLD | COLOR_PAIR(2));
printw("Bold yellow on black");
attroff(A_BOLD | COLOR_PAIR(2));

8. Keypad & Special Keys


The keypad() function you call during initialization tells ncurses to translate escape sequences from special keys (arrows, F-keys) into symbolic constants.


// MUST be called on stdscr during initialization
keypad(stdscr, TRUE);


Without keypad(stdscr, TRUE), pressing the Up arrow gives you raw escape sequences (ESC [ A) instead of KEY_UP.



  
Critical

  Always call keypad(stdscr, TRUE) or your editor won't respond to arrow keys, function keys, or any special key. This is the most common beginner error.

9. Echo, Cursor & Visibility


Echo Control


noecho();     // Don't show typed characters (default after noecho)
echo();       // Show typed characters as you type


For an editor, you almost always want noecho(). For a command-line input (like a search bar), you temporarily enable echo.


Cursor Control


 curs_set(1);     // Visible cursor (0=hidden, 1=block, 2=underline)
move(y, x);       // Move cursor to (y, x)


Nodelay & Timeout


// nodelay: getch returns ERR immediately if no key
nodelay(stdscr, TRUE);

// timeout: getch blocks for ms milliseconds, then returns ERR
timeout(50);      // 50ms timeout

// Blocking (default)
nodelay(stdscr, FALSE);
timeout(-1);       // Negative = infinite wait










10. Event Loop Pattern


Here's the standard pattern for a TUI editor's main loop:


// After initscr() and configuration...

bool running = true;
while (running) {
    // 1. Render the current state
    draw_screen();

    // 2. Get input (non-blocking for polling, or blocking for waiting)
    timeout(-1);     // Block until key press
    int key = getch();

    // 3. Handle the key
    switch (key) {
        case KEY_UP:
            move_cursor_up();
            break;
        case KEY_DOWN:
            move_cursor_down();
            break;
        case 'q':
            running = false;
            break;
        case KEY_RESIZE:
            handle_resize();
            break;
        // ... more key handlers ...
    }
}

endwin();

11. Terminal Resize


ncurses sends KEY_RESIZE when the user resizes the terminal. You should handle this:


// In your event loop:
case KEY_RESIZE:
    handle_resize();
    break;

// Resize handler:
void handle_resize() {
    // Wipe and recreate all windows to new size
    delwin(status_bar);    // Delete old windows
    delwin(editor_win);
    erase();

    // Recreate with new dimensions
    int rows = LINES;
    int cols = COLS;
    status_bar = newwin(1, cols, rows - 1, 0);
    editor_win = newwin(rows - 2, cols, 0, 0);
}



  
Tip

  On macOS, the terminal may also send a SIGWINCH signal. You can install a signal handler to set a flag, which your event loop checks alongside KEY_RESIZE.

Key Design Decisions for Your Editor


    
  
  1. Use wide-character functions: wgetch(), waddwstr(), wprintw() for Unicode support.
    2. 
  
  2. One window for stdscr: Don't create too many sub-windows. Use mvprintw on stdscr directly for simplicity.
    3. 
  
  3. Separate render from logic: Your command logic shouldn't know about ncurses. Use an abstraction layer.
    4. 
  
  4. Handle KEY_RESIZE: Always handle terminal resize to avoid display corruption.
    5. 
  
  5. Use timeout(-1) for blocking input in your main loop, or timeout(50) for polling if you need periodic updates.

13. Building Your Editor — Complete Skeleton


Here's a complete minimal editor using ncurses. This is a starting point you can extend:


#include <ncurses.h>
#include <vector>
#include <string>

struct Buffer {
    std::vector<std::string> lines;
    int cy = 0, cx = 0;

    void insert_char(char c) {
        if (cy < lines.size()) {
            lines[cy].insert(cx, 1, c);
        } else {
            lines.push_back("1", c);
        }
        ++cx;
    }

    void backspace() {
        if (cx > 0) {
            lines[cy].erase(cx - 1, 1);
            --cx;
        }
    }

    void move_up() { if (cy > 0) --cy; }
    void move_down() {
        if (cy < (int)lines.size() - 1) ++cy;
    }
};

void render(const Buffer& b) {
    erase();
    for (int i = 0; i < LINES - 1; ++i) {
        if (i < (int)b.lines.size())
            mvprintw(i, 0, "%s", b.lines[i].c_str());
    }
    mvprintw(LINES - 1, 0, "L:%d C:%d,%d",
        (int)b.lines.size(), b.cy, b.cx);
    refresh();
}

int main() {
    initscr();
    cbreak();
    noecho();
    keypad(stdscr, TRUE);
    curs_set(1);
    timeout(-1);

    Buffer buf;
    buf.lines.push_back("Welcome to your editor!");
    buf.lines.push_back("Press q to quit.");

    while (true) {
        render(buf);
        int key = getch();

        switch (key) {
            case KEY_UP:    buf.move_up();    break;
            case KEY_DOWN:  buf.move_down();  break;
            case KEY_LEFT:  if (buf.cx > 0) --buf.cx; break;
            case KEY_RIGHT: ++buf.cx;          break;
            case KEY_BACKSPACE: buf.backspace(); break;
            case KEY_DC:    if (buf.cx < buf.lines[buf.cy].size())
                             buf.lines[buf.cy].erase(buf.cx, 1);
                            break;
            case KEY_ENTER:
            case '\n':
                {
                    std::string before = buf.lines[buf.cy].substr(0, buf.cx);
                    std::string after  = buf.lines[buf.cy].substr(buf.cx);
                    buf.lines[buf.cy] = before;
                    buf.lines.insert(buf.lines.begin() + buf.cy + 1, after);
                    ++buf.cy;
                    buf.cx = 0;
                }
                break;
            case KEY_RESIZE:;   // ignore break;
            case 'q': return 0;
            default:
                if (key >= 32 and key < 127)
                    buf.insert_char(char(key));
                break;
        }
    }
}

Common Mistakes


    
  
  • Forgetting refresh(): ncurses buffers output. Without refresh, nothing appears on screen.
    • 
  
  • Forgetting keypad(stdscr, TRUE): Arrow keys and F-keys won't work.
    • 
  
  • Coordinates are (y, x): Row first, then column. Not (x, y).
    • 
  
  • Not calling endwin() on crash: Terminal is left broken. Use a destructor or atexit handler.
    • 
  
  • Using printw after a keypress: printw changes the cursor position. If you're drawing in a specific spot, you may need to reposition.
    •
